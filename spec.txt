# CanisUIForge – Technical & Functional Specification

---

# 1. Overview

## 1.1 Product Name

**CanisUIForge**

## 1.2 Purpose

CanisUIForge is a code generation tool that consumes a Swagger/OpenAPI definition and a Contracts source to generate fully structured UI applications.

It supports generating:

* Blazor applications
* .NET MAUI applications
* Both simultaneously

The generated applications follow:

* layered architecture
* reusable component design
* centralized styling
* strict engineering standards (DRY, SOLID)
* safe regeneration patterns
* full test scaffolding (unit + UI automation)

---

# 2. Core Design Philosophy

## 2.1 Contracts-First Generation

Swagger is used for:

* endpoint discovery
* route structure
* HTTP methods
* controller grouping

Contracts are used for:

* request models
* response models
* namespaces
* type correctness

### Rule

The generator must **not generate DTOs from Swagger** when contracts are provided.

---

## 2.2 Generated vs Manual Code Separation

All generated code must:

* be placed in dedicated generated folders or files
* be safe to overwrite
* never overwrite manual code

### Pattern

* `*.Generated.cs`
* `Generated/` folders
* partial classes for extension

---

## 2.3 Engineering Standards

Generated code must:

* follow DRY
* follow SOLID
* use explicit types (no `var`)
* use global usings
* avoid unofficial abbreviations
* never use `ConfigureAwait`
* use proper async/await patterns
* be readable and production-quality
